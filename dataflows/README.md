# 数据层模块

统一的数据获取接口，支持多源自动回退和公共数据缓存。

## 文件说明

| 文件 | 说明 |
|------|------|
| `interface.py` | 统一数据路由，对外提供单一入口，内部按数据需求分发到不同适配器，并拦截公共数据走缓存 |
| `akshare_adapter.py` | AKShare 适配器，封装所有 AKShare API 调用，支持多源回退 |
| `market_cache.py` | 市场公共数据缓存层（内存 + 磁盘双层），避免批量分析时重复拉取 |

## 多源回退策略

当主数据源不可用时，自动切换到备选源：

| 数据类型 | 主源 | 回退1 | 回退2 |
|----------|------|-------|-------|
| 个股日K线 | 东方财富 | 新浪财经 | 腾讯财经 |
| 实时行情 | 东方财富 | 新浪财经 | 个股K线末日 |
| 市场情绪 | 乐咕乐股 | 新浪个股列表 | — |
| 行业板块 | 东方财富 | 同花顺 | — |

## 公共数据缓存

批量分析时，以下数据对所有个股都相同，缓存后避免重复拉取：

| 缓存项 | 方法名 | 涉及分析师 | 缓存策略 |
|--------|--------|-----------|----------|
| 市场情绪 | `get_market_sentiment` | 技术面 + 情绪面 + 政策面 | 内存 + 磁盘 |
| 北向资金 | `get_north_flow` | 技术面 + 政策面 | 内存 + 磁盘 |
| 行业板块 | `get_sector_boards` | 技术面 + 政策面 | 内存 + 磁盘 |
| 概念板块 | `get_concept_boards` | （预留） | 内存 + 磁盘 |

- **内存缓存**：当前会话内即时命中，零 IO 开销
- **磁盘缓存**：按交易日分文件存储（`~/.astock_agent/market_cache/`），跨会话复用
- **自动拦截**：`interface.py` 的 `route_to_vendor()` 自动识别公共方法并走缓存，对上层透明

## 设计思路

- 借鉴 TradingAgents 的统一数据接口理念
- 针对 push2.eastmoney.com 在国内部分网络环境不稳定问题，设计了完整的多源回退链
- 公共数据缓存层解决批量分析时的重复拉取问题，25 只股票从最多 175 次请求降至 4 次
