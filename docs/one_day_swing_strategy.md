# 一日游策略设计：快速反馈 + Agent OPT 自动化

> 核心思想：**一日游不是为了让单次收益最大化，而是为了让反馈样本积累速度最大化。**

---

## 1. 为什么是一日游

### 1.1 问题：长线策略的反馈鸿沟

```
长线策略：
Day 0: 分析 → Buy → 持有 2-4 周 → 才知道对错
                     ↑
               反馈周期：10-20 个交易日

月样本量：25 支 × 1 轮 = 25 条反馈
```

反馈太慢，意味着：
- 你无法快速判断策略对不对
- Agent 的 prompt 怎么调？等一个月才知道效果
- 参数怎么优化？一个月只有一次实验机会
- 策略迭代速度被「等反馈」锁死

### 1.2 解决：一日游压缩反馈周期

```
一日游策略：
Day 0: 收盘分析 → Day 1: 开盘买入 → Day 2: 收盘平仓
                                          ↓
                                    Day 2 收盘即知结果

反馈周期：2 天（vs 长期 10-20 天）

周样本量：25 支 × 5 天 = 125 条反馈/周
月样本量：25 支 × 20 天 = 500 条反馈/月
```

**一个月的一日游 = 一年半的长线反馈量**。

### 1.3 反馈密度的意义

| 指标 | 长线策略 | 一日游策略 |
|------|:--:|:--:|
| 单次反馈周期 | 2-4 周 | **2 天** |
| 月可迭代轮次 | 1-2 次 | **15-20 次** |
| 月样本量 | ~25 条 | **~500 条** |
| 策略调优速度 | 以月计 | **以天计** |
| A/B 测试可行性 | 差 | **好**（每周可对比不同参数） |

核心公式：

```
策略进化速度 = 反馈质量 × 反馈数量 × 反馈频率
               ↑           ↑           ↑
            有监督信号   大样本    一日游天然优势
```

## 2. Agent OPT（自动化优化闭环）

### 2.1 什么是 Agent OPT

`Agent OPT` 不是调模型参数（fine-tuning），而是**基于回测数据自动优化 Agent 的 System Prompt + 运行参数**，让多 Agent 系统在「命中率/踏空率/Buy信号产出率」三个指标上达到最优平衡。

### 2.2 优化维度

| 优化对象 | 当前配置 | 优化方向 |
|----------|----------|----------|
| Bull Researcher 风险偏好 | 偏保守 | 增加日内动量权重，减少长期估值权重 |
| Bear Researcher 质疑强度 | 过强 | 降低「涨不到1%」的阈值敏感性 |
| PM 决策阈值 | conf≥50%才Buy | 探索 conf≥40% 的命中率变化 |
| 辩论轮数 | 1 轮 | 测试 2 轮的决策质量 vs 耗时 |
| 流动性门槛 | 日成交额≥1亿 | 测试 5000万 的影响 |
| Debater 权重 | 三方平等 | 测试 Bull:Neutral:Conservative=4:3:3 |

### 2.3 OPT 流程

```
┌──────────────────────────────────────────────┐
│                  Agent OPT 循环               │
│                                              │
│  1. 跑一批分析 (25支，Day0-当前日)            │
│           ↓                                  │
│  2. 等待 Day1 收盘 → 回测验证                 │
│           ↓                                  │
│  3. 汇总指标：命中率/踏空率/Buy信号率          │
│           ↓                                  │
│  4. 判断是否需要调整？                         │
│     ├─ 误判 > 0：收紧风险                     │
│     ├─ 踏空 > 30%：放宽信号出产                │
│     ├─ Buy信号率 < 5%：太保守，调高 Bull 权重   │
│     └─ 命中率 + 踏空率 均优：保持              │
│           ↓                                  │
│  5. 自动修改 prompt/参数，下一轮生效           │
│           ↓                                  │
│  6. 回到步骤 1                                │
└──────────────────────────────────────────────┘
```

### 2.4 回测指标体系

| 指标 | 计算 | 含义 | 红线 |
|------|------|------|:--:|
| **Buy 命中率** | HIT / (HIT+MISS) | Buy信号可靠性 | <50% ❌ |
| **总准确率** | (HIT+AVOID) / 全部 | 整体判断正确率 | <55% ❌ |
| **踏空率** | STEP / 全部 | 该买没买的比例 | >40% ❌ |
| **误判率** | MISS / 全部 | 买了但跌的比例 | >0% ❌ |
| **Buy 信号率** | Buy数 / 全部 | 每日产出Buy的比例 | <5% 🟡太保守 |
| **净收益率** | Σ(命中股净收益) / Buy数 | Buy信号平均净收益 | <0.5% ❌ |
| **夏普比率(模拟)** | 平均收益/标准差 | 风险调整后收益 | <0.5 ❌ |

### 2.5 自动调优规则（第一版）

```python
# 伪代码
def auto_tune(metrics: dict, config: dict) -> dict:
    """基于回测指标自动调整配置"""

    buy_rate = metrics["buy_count"] / metrics["total"]
    miss_rate = metrics["miss"] / metrics["total"]

    # 规则1: 太保守 → 降低 Bull 门槛
    if buy_rate < 0.05 and miss_rate == 0:
        # 连续 3 天零误判但 Buy<5%，说明太保守
        config["bull_weight"] += 0.15
        config["pm_confidence_threshold"] = max(0.40, config["pm_confidence_threshold"] - 0.05)

    # 规则2: 有误判 → 提高门槛
    if metrics["miss"] > 0:
        config["pm_confidence_threshold"] = min(0.80, config["pm_confidence_threshold"] + 0.10)
        config["bear_weight"] += 0.10

    # 规则3: 踏空太多 → 放宽信号
    step_rate = metrics["step"] / metrics["total"]
    if step_rate > 0.35:
        config["pm_confidence_threshold"] = max(0.35, config["pm_confidence_threshold"] - 0.05)

    return config
```

## 3. 为什么一日游特别适合 Agent OPT

### 3.1 样本量碾压

```
一日游: 25支/天 → 500支/月 → 每 2 天就有 25 个新样本喂给 OPT
长线:   25支/月 → 25支/月   → 要等一个月才知道参数调得对不对
```

500 个样本已经足够做统计显著性检验。一条 prompt 好不好，一两天就能看出来。

### 3.2 信号纯净

一日游持有 24 小时，**噪音更少**：
- 没有「长期基本面变化」的干扰
- 没有「政策窗口期延后」的不确定性
- 涨跌完全由日内动量+隔夜消息驱动 → Agent 的判断维度简单、可归因

### 3.3 快速 A/B 测试

```
周一：用 prompt-A 跑 25 支
周二：用 prompt-B 跑 25 支
周三：两个版本的命中率一目了然 → 周四切到更优版本
```
